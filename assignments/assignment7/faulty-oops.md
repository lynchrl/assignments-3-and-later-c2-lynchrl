## Analysis

The Oops indicates a kernel NULL pointer dereference at virtual address `0000000000000000`. The error occurs in the `faulty_write` function, which is part of the `faulty` module. The fault is at offset `0x10` from the start of the `faulty_write` function.

For reference the `faulty_write` function is defined as follows:

```c
ssize_t faulty_write (struct file *filp, const char __user *buf, size_t count,
		loff_t *pos)
{
	/* make a simple fault by dereferencing a NULL pointer */
	*(int *)0 = 0;
	return 0;
}
```

From this, we can see the explicit deferencing of a NULL pointer.
